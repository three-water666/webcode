# v0.8.0 (2026-05-22)

### 新增
- **Keepalive 浏览器模式**：新增 isolated Edge/Chrome 和用户配置 Chrome/Edge keepalive 启动模式，并为隔离配置支持内置 bridge 扩展。 (#24)
- **HITL 注意力提醒**：新增浏览器注意力请求和任务完成通知，让长任务或等待审批的工作能在非活动页面外提示。 (#27)
- **快速启动流程**：启动 gateway 前要求存在 VS Code 工作区，并在启动后立即打开 AI 启动菜单。

### 改进
- **浏览器打包**：VSIX 打包和浏览器 zip 发布复用 browser bridge 构建，并将 bridge 扩展打入 VS Code 包以支持隔离启动。 (#24)
- **默认浏览器模式**：quick launch 和文档默认推荐 isolated Edge，减少单独安装浏览器扩展的需要。
- **提示词指引**：澄清任务完成通知时机、顺序工具执行、运行时兜底默认值和浏览器模式行为。

### 修复
- **顺序工具执行**：将捕获到的浏览器工具调用排队，并在 HITL 审批和后端响应完成后再执行下一个工具。 (#26)
- **Keepalive 启动可靠性**：允许隔离浏览器配置加载扩展，加固 keepalive 启动错误处理，并在浏览器过早退出时回退。 (#24, #25)
- **完成通知**：避免丢弃窗口状态不确定时的完成通知，并在虚拟工具被调用时始终发送完成通知。 (#27)
- **命令审批排序**：按覆盖范围排序命令审批规则，让更具体和更宽泛的 scope 以可预期方式评估。

### 工程
- **启动基础设施**：新增浏览器进程启动、进程检测、浏览器扩展复制和 prepublish 辅助逻辑，用于打包后的隔离浏览器支持。 (#24)
- **发布脚本**：刷新 Windows 和 shell 发布脚本，让各发布产物一致复用 browser build。 (#24)
