# v0.7.0 (2026-05-21)

### 新增
- **本地 VS Code 工具**：新增内置工作区工具，可通过 gateway 读取、写入和搜索代码，列出项目规则和 skills，运行终端并执行命令。 (#16)
- **Claude 支持**：新增 Claude 内置平台，沿用现有 AI 站点的 bridge 接入流程。 (#18)
- **严格工具校验**：新增更严格的参数 schema 校验，在执行前拒绝格式错误或未知参数的工具调用。 (#15)

### 改进
- **文件工具默认行为**：代码检查优先使用内置文件工具，默认截断过大的 `read_file` 输出，并补充行号行为说明。 (#20)
- **Gateway 架构**：将扩展激活、gateway 路由、bridge 路由和服务管理拆分为更聚焦的模块，便于维护。 (#16, #22)
- **浏览器 Bridge UI**：将内容请求处理、审批弹窗、日志 UI、自动发送、结果投递和可视状态辅助逻辑拆分到独立模块。 (#22)
- **MCP 文档**：新增英文和中文 MCP 服务器配置指南，并刷新初始化和超长提示文案。 (#16, #19)

### 修复
- **命令审批安全**：加固命令审批规则处理，并规范审批匹配逻辑。 (#22)
- **文件读取可靠性**：流式处理范围读取，保留前缀中的非法 UTF-8 字节，夹紧返回范围，并让前缀读取更可靠。 (#20)
- **工具安全边界**：加固路径边界，将带括号的文件搜索视为字面量，拒绝未知参数，并为第三方 MCP 工具名加命名空间。 (#15, #16)
- **HITL 流程**：保持拒绝输入框聚焦，避免将 HITL host 误识别为聊天输入，并保持 host 不可编辑。 (#17, #21)
- **运行稳定性**：通过 bridge 传递 gateway 工具错误，允许协议错误后的重试，在工作区短暂缺失时保留 skill 缓存，并防止终端会话成功状态过期泄漏。 (#15, #16)
- **日志边界**：拖动或从最小化状态恢复时，让浮动日志面板保持在视口范围内。 (#16)

### 工程
- **模块化重构**：将大型 gateway、扩展、bridge UI 和内容脚本拆分为更小模块，不改变发布产物边界。 (#16, #22)
- **测试**：新增 path arguments、`read_file` 行为和文件搜索行为覆盖。 (#16, #20)
