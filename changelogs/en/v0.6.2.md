# v0.6.2 (2026-04-25)

### Features
- **Project Branding**: Renamed the browser extension, VS Code gateway, package metadata, documentation, and release artifacts around the `webcode` product name.
- **Initialization Prompt Shortcut**: Added in-chat `/webcode` and `@webcode` triggers that ask before replacing the trigger word with the initialization prompt, without auto-sending the message.
- **Tool Call Status Badges**: Added localized status badges to captured tool-call JSON blocks so running, completed, and error states are easier to identify.

### Improvements
- **Initialization Guidance**: Clarified the initialization prompt and `webcode_init` result text so models are less likely to repeat the initialization command after setup.
- **Docs**: Updated README usage instructions and browser extension documentation to describe the new `/webcode` and `@webcode` initialization flow.

### Fixes
- **Browser Extension Packaging**: Exposed generated browser extension chunks so content scripts can load built assets correctly.
- **Initialization Trigger Matching**: Fixed `/webcode` and `@webcode` detection when the trigger is adjacent to non-ASCII task text, while still avoiding URL and handle-style false positives.
- **HITL Queue**: Fixed repeated confirmation prompts for queued commands after the first matching command has been permanently allowed.
