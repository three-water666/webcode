# v0.6.3 (2026-05-13)

### Features
- **Command Safety**: Added risk detection for shell commands so dangerous command patterns can be rejected before execution.
- **Tool Call Feedback**: Added a running-state loading animation and aligned ChatGPT tool-call status badges for clearer in-page execution feedback.

### Improvements
- **Command Execution**: Unified command shell handling and improved terminal process cleanup so stopped sessions terminate child processes more reliably.
- **Browser Bridge**: Improved browser extension development stability, hot reload behavior, and auto-send DOM query typing.
- **Tool Call Parsing**: Improved recovery for malformed model tool-call JSON and deduplication for tool calls that do not include a request ID.

### Fixes
- **Auto Send**: Fixed duplicate browser auto-send behavior.
- **Reasoning Blocks**: Ignored DeepSeek and GLM reasoning/tool-thinking blocks so they are not treated as executable tool calls.
- **VS Code Diagnostics**: Cleaned up extension debug diagnostics and related development configuration.

### Engineering
- **Release Workflow**: Added the tag-triggered GitHub release workflow, split bilingual changelogs by version, and documented the Windows PowerShell release build path.
- **Quality Checks**: Strengthened ESLint coverage and applied automated lint fixes across the workspace.
