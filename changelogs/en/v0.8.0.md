# v0.8.0 (2026-05-22)

### Features
- **Keepalive Browser Modes**: Added isolated Edge/Chrome and user-profile Chrome/Edge keepalive launch modes, with bundled bridge extension support for isolated profiles. (#24)
- **HITL Attention**: Added browser attention requests and task completion notifications so long-running or approval-blocked work can surface outside the active page. (#27)
- **Quick Launch Flow**: Require a VS Code workspace before starting the gateway, then open the AI launch menu immediately after startup.

### Improvements
- **Browser Packaging**: Reuse the browser bridge build for both VSIX packaging and browser zip releases, and bundle the bridge extension into the VS Code package for isolated launches. (#24)
- **Default Browser Mode**: Prefer isolated Edge for quick launch and documentation, reducing the need for a separately installed browser extension.
- **Prompt Guidance**: Clarify completion notification timing, sequential tool execution, runtime fallback defaults, and browser mode behavior.

### Fixes
- **Sequential Tool Execution**: Queue captured browser tool calls and wait for HITL approval and backend responses before starting the next tool. (#26)
- **Keepalive Launch Reliability**: Allow extensions in isolated browser profiles, harden keepalive launch errors, and fall back when a browser exits early. (#24, #25)
- **Completion Notifications**: Avoid dropping uncertain completion notifications and always send completion notifications when the virtual tool is called. (#27)
- **Command Approval Ordering**: Order command approval scopes by breadth so more specific and broader scopes are evaluated predictably.

### Engineering
- **Launch Infrastructure**: Add browser process launching, process detection, browser extension copy, and prepublish helpers for packaged isolated browser support. (#24)
- **Release Scripts**: Refresh Windows and shell release scripts to package the shared browser build consistently across artifacts. (#24)
