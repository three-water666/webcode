# v0.7.0 (2026-05-21)

### Features
- **Local VS Code Tools**: Added built-in workspace tools for reading, writing, and searching code, listing project rules and skills, running terminals, and executing commands through the gateway. (#16)
- **Claude Support**: Added Claude as a built-in platform with the same bridge setup used by the existing AI sites. (#18)
- **Strict Tool Validation**: Added stricter argument schema validation so malformed or unknown tool calls are rejected before execution. (#15)

### Improvements
- **File Tool Defaults**: Prefer built-in file tools for code inspection and truncate oversized `read_file` output by default while documenting line-number behavior. (#20)
- **Gateway Architecture**: Split extension activation, gateway routing, bridge routing, and service management into focused modules for easier maintenance. (#16, #22)
- **Browser Bridge UI**: Split content request handling, approval dialogs, logger UI, auto-send, result delivery, and visual status helpers into dedicated modules. (#22)
- **MCP Documentation**: Added English and Chinese MCP server configuration guides and refreshed initialization and oversize prompt copy. (#16, #19)

### Fixes
- **Command Approval Safety**: Hardened command approval rule handling and normalized approval matching. (#22)
- **File Reading Reliability**: Streamed ranged reads, preserved invalid UTF-8 bytes in prefixes, clamped returned ranges, and kept prefix reads reliable. (#20)
- **Tool Safety Boundaries**: Hardened path boundaries, treated bracketed file searches as literals, rejected unknown arguments, and namespaced third-party MCP tools. (#15, #16)
- **HITL Flow**: Kept rejection input focused, avoided matching the HITL host as a chat input, and kept the host non-editable. (#17, #21)
- **Runtime Stability**: Propagated gateway tool errors through the bridge, allowed retries after protocol errors, kept skill cache during workspace gaps, and prevented stale terminal-session success leaks. (#15, #16)
- **Logger Bounds**: Kept the floating logger constrained while dragging or restoring from a minimized state. (#16)

### Engineering
- **Modular Refactors**: Split large gateway, extension, bridge UI, and content scripts into smaller modules without changing release artifact boundaries. (#16, #22)
- **Tests**: Added coverage for path arguments, `read_file` behavior, and file search behavior. (#16, #20)
