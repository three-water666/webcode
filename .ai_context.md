## 1. 项目概览 (WebMCP)

**目标**: 通过 MCP 协议，将基于 Web 的 AI (ChatGPT/DeepSeek) 连接到本地 VS Code 环境，赋予其文件读写、代码执行和 Git 管理能力。

**核心架构**: `Browser AI` <-> `Browser Extension (Bridge)` <-> `VS Code Extension (Gateway)` <-> `Local Tools`

---

## 2. 组件详解

### 🔌 A. VS Code 插件 (`mcp-gateway-vscode`)
**角色**: MCP Server / Host (聚合器 & 网关)

- **核心功能**:
  - **本地服务器**: 启动 Express Server (默认端口 `34567`)。
    - **端口自增**: 若端口被占用，自动 +1 重试（最大 20 次），支持多实例并存。
  - **工具聚合**: 管理并启动子 MCP 服务（如 `@modelcontextprotocol/server-filesystem`, `git`, `fetch`），将它们的能力合并暴露。
  - **安全握手**: 
    - 每次启动生成随机 `UUID Token`。
    - 提供 `/bridge` 中转页，自动携带 Token 和端口信息唤起浏览器，实现零配置连接。
  - **配置管理**: 通过 `settings.json` 配置子服务参数（如 GitHub Token、Docker 容器参数）。

- **交互体验 (Status Bar)**:
  - **Offline**: 点击状态栏 -> 弹出 "Turn On" 菜单。
  - **Online**: 点击状态栏 -> 弹出 "Quick Launch" 菜单（一键打开 DeepSeek/ChatGPT）。
  - **Browser Selector**: 支持按站点配置默认浏览器（如强制 DeepSeek 用 Edge 打开），或使用 `Custom Launch` 手动选择。

### 🧩 B. 浏览器插件 (`mcp-bridge-browser`)
**角色**: MCP Client / Bridge (注入器 & 客户端)

#### 1. 核心逻辑 (Content Scripts)
- **模块化架构**:
  - `content.js`: **大脑**。负责轮询 DOM 消息、解析 AI 响应、调度任务队列。
  - `modules/utils.js`: **基建**。提供悬浮日志 (Logger)、多语言 (i18n) 支持。
  - `modules/ui.js`: **交互**。实现 HITL 审批弹窗 (Shadow DOM)、自动发送、输入框回填。
- **持久化**: `background.js` 负责在插件更新时保护用户配置不被覆盖，并处理安全熔断（URL 变更时切断连接）。

#### 2. 交互界面 (Popup)
- **三态视图切换**:
  1. **Connected (已连接)**: 显示当前绑定的端口号、Token 状态，提供 "Show Logs" 开关。
  2. **Available (快速连接)**: 在白名单网站上，扫描本地 `storage` 中已活跃的 Gateway Session，显示 "🔗 Connect to 34567" 按钮，一键恢复连接。
  3. **Disconnected (未连接)**: 在未授权网站（如 Baidu）或无本地服务时显示，提示用户先去 VS Code 启动服务。

#### 3. 设置页面 (Options)
- **🛡️ Human-in-the-Loop 设置**: 动态列出所有可用工具（从 Content Script 自动缓存），用户可勾选“敏感工具”（如 `git_commit`），触发人工审批拦截。
- **提示词管理**: 自定义 `System Prompt` (初始设定)、`Training Hint` (定期复训) 和 `Error Hint` (纠错提示)。
- **DOM 选择器**: 高级配置，允许用户针对不同 AI 网站 (DeepSeek/ChatGPT) 微调 DOM 抓取规则。

## 3. 关键特性机制

### 🛡️ Human-in-the-Loop (HITL) 人工审批
- **机制**: `content.js` 在执行工具前检查 `protected_tools` 名单。
- **交互**: 命中拦截时，通过 **Shadow DOM** 弹出隔离窗口，展示工具参数。
- **流程**: 
  - **批准**: 立即释放请求，自动聚焦输入框并发送。
  - **拒绝**: 用户填写拒绝理由，系统生成包含 `User rejected` 的错误报文回传给 AI。

### ⚡ 并发批处理队列 (Concurrency Queue)
- **痛点**: 解决 AI 一次性调用多个工具（如 1个快 + 1个慢 Fetch）导致的回填乱序或漏发问题。
- **逻辑**: `resultBuffer` 暂存所有结果 -> 等待当前轮次 **所有** 工具完成 -> 按 DOM 顺序拼接 -> 一次性回填。

### 🔄 健壮的自动发送 (Robust Auto-Send)
- **焦点抢占**: 在点击发送前强制 `inputEl.focus()`，绕过浏览器的后台失焦限制。
- **事件模拟**: 触发 `input/change` 事件，确保 React/Vue 框架能检测到文本变化。
- **重试与报警**: 包含 5 次指数退避重试，失败则弹出系统通知。
