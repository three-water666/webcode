## 1. Project Overview (WebMCP)

**Goal**: Connect Web-based AI (ChatGPT/DeepSeek) to Local VS Code via MCP Protocol.

### 🏗 Structure
- **Root**: Contains build scripts (`build_release.sh`) and global docs.
- **`/mcp-gateway-vscode`** (VS Code Extension):
  - Acts as the MCP Server (Express).
  - **Port**: Default `34567` (Dynamic range 30000-40000).
  - **Key File**: `src/gateway.ts` (Handles `/bridge` page and WebSocket/HTTP).
- **`/mcp-bridge-browser`** (Browser Extension):
  - Injects into AI pages to intercept tool calls.
  - **Key Files**:
    - `content.js`: Main logic. Reads config from Storage (priority) or Defaults.
    - `background.js`: Persistent Service Worker. Initializes defaults from `.md` files.
    - `config.js`: **Single Source of Truth** for default DOM Selectors.
    - `options.html/js`: Settings page for user customization.
  - **i18n Resources** (Web Accessible):
    - `prompt.md` / `prompt_zh.md`: System prompts (Default).
    - `train.md` / `train_zh.md`: Periodic training hints (Default).
    - `error_hint.md` / `error_hint_zh.md`: Format error warnings (Default).

## 2. Key Mechanisms (Do Not Break)

### ⚙️ Configuration System (Updated)
- **VS Code Gateway**:
  - `mcpGateway.aiSites`: Defines AI platforms. Now supports per-site `browser` configuration (e.g., force "Edge" for DeepSeek).
  - `mcpGateway.browser`: Global fallback browser.
  - *Removed*: `mcpGateway.browserRules` (functionality merged into `aiSites`).
- **Browser Bridge**:
  - **Defaults**: Stored in `config.js` (Selectors) and `.md` files (Prompts/Hints).
  - **User Config**: Stored in `chrome.storage.sync` (Selectors) and `chrome.storage.local` (Texts).
  - **Hot Reload**: `content.js` listens to `chrome.storage.onChanged`.

### 🔄 Cross-Promotion (互推逻辑)

### 🛡️ Security Hardening (重要)
- **Gateway Server**: Token-based authentication is now protected by **Strict Origin Whitelisting**.
  - **Whitelist Source**: Dynamically built from `mcpGateway.aiSites` addresses + default local domains (`localhost`, `127.0.0.1`).
  - **Defense**: Prevents Cross-Origin attacks (e.g., CSRF/Token leakage via browser history) even if the authentication token is known by an external attacker.

- **Gateway (VS Code)**: The `/bridge` page checks `data-extension-installed` attribute.
- **Bridge (Browser)**: Popup guides user based on installation status.

### 🧠 AI Memory Maintenance
- **Periodic Training**: `content.js` counts tool calls. Every 5th call, it injects a `system_note`.
  - **Logic**: Uses `train.md` (or `_zh`) from Storage.

## 5. Implementation Notes (Critical Logic)

### 🔧 Browser Bridge (`content.js` / `background.js`)
- **Security Fuse**: `background.js` monitors URL changes. If navigating outside allowed hosts, the session is IMMEDIATELLY revoked.
- **Popup Logic (Three-State)**: 1. Connected (Status) 2. Disconnected (Guide) 3. Available (List of active gateways). 
  - **Security Constraint**: States 1 & 3 are ONLY allowed on `Allowed Whitelist` domains. Other domains MUST show State 2 (Disconnected) and block gateway scanning.
  - **Whitelist Definition**: Union of `host_permissions` AND `content_scripts.matches` (to include specific localhost ports).
- **Virtual Tools**: `content.js` intercepts `list_tools` to inject browser-side only tools (e.g., `task_completion_notification`).
  - **Logic**: Processed locally. Sends system notification and **exits immediately** (no result backfill to chat) to keep history clean.
- **JSON Fault Tolerance**: Implements **Stabilization Detection** (3s timeout). If a JSON block is invalid and stops updating (stuck), it triggers a **Red Border** and System Notification.
- **Token Tracker**: Real-time context usage estimation with **Incremental Persistence**. Uses URL-based ChatIDs to track usage across sessions/refreshes. Auto-prunes data > 10 days.
- **Input Backfill Logic**: MUST use `document.execCommand('insertText')` to prevent `contenteditable` double-spacing issues.
- **VS Code UI**: Status bar now supports `$(sync~spin)` loading state during startup/restart.
- **Retry Mechanism**: Implements **Debounce** to prevent race conditions. Uses unified `SHOW_NOTIFICATION` channel for all alerts (Task done, JSON Error, Auto-send failed).
- **Selector Strategy**: Prefer SVG path detection (`:has(path[...])`) over unstable CSS classes for critical buttons (like DeepSeek Send).

### 📦 Build Scripts
- **Windows PowerShell**: Use ASCII tags (`[OK]`) to prevent Mojibake.
