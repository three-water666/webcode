## 1. 项目概览 (WebMCP)

**目标**: 通过 MCP 协议，将基于 Web 的 AI (ChatGPT/DeepSeek) 连接到本地 VS Code 环境，赋予其文件读写、代码执行和 Git 管理能力。

**核心架构**: `Browser AI` <-> `Browser Extension (Bridge)` <-> `VS Code Extension (Gateway)` <-> `Local Tools`

---

## 2. 工程架构 (Engineering)

### 🏗️ Monorepo 结构 (pnpm workspace)
项目采用 Monorepo 模式管理，根目录包含 `pnpm-workspace.yaml`。

- **根目录**: 统一依赖管理，共享配置。
- **`shared/`**: (New) 存放两端共用的类型定义 (`types.ts`) 和协议常量，确保通信一致性。
- **`mcp-gateway-vscode/`**: VS Code 插件项目 (Webpack + TS)。
- **`mcp-bridge-browser/`**: 浏览器插件项目 (Vite + CRXJS + TS)。
  - **构建工具**: Vite + @crxjs/vite-plugin (支持 HMR 热更新)。
  - **语言**: TypeScript (全量类型覆盖)。
  - **UI**: 原生 DOM 操作 (保持轻量，已封装为 `src/modules/ui.ts`)。

## 3. 组件详解

### 🔌 A. VS Code 插件 (`mcp-gateway-vscode`)
**角色**: MCP Server / Host (聚合器 & 网关)

- **核心功能**:
  - **本地服务器**: 启动 Express Server (默认端口 `34567`)。
    - **端口自增**: 若端口被占用，自动 +1 重试（最大 20 次），支持多实例并存。
  - **工具聚合**: 管理并启动子 MCP 服务（如 `@modelcontextprotocol/server-filesystem`, `git`, `fetch`），将它们的能力合并暴露。
  - **安全握手**: 
    - 每次启动生成随机 `UUID Token`。
    - 提供 `/bridge` 中转页，自动携带 Token 和端口信息唤起浏览器，实现零配置连接。
  - **配置管理**: 通过 `settings.json` 配置子服务参数（如 GitHub Token、Docker 容器参数）。

- **交互体验 (Status Bar)**:
  - **Offline**: 点击状态栏 -> 弹出 "Turn On" 菜单。
  - **Online**: 点击状态栏 -> 弹出 "Quick Launch" 菜单（一键打开 DeepSeek/ChatGPT）。
  - **Browser Selector**: 支持按站点配置默认浏览器，或使用 `Custom Launch` 手动选择。

### 🧩 B. 浏览器插件 (`mcp-bridge-browser`)
**角色**: MCP Client / Bridge (注入器 & 客户端)

#### 1. 核心逻辑 (Content Scripts)
- **模块化架构 (src/)**:
  - `content/main.ts`: **大脑**。负责轮询 DOM 消息、解析 AI 响应、调度任务队列。
  - `modules/utils.ts`: **基建**。提供悬浮日志 (Logger)、多语言 (i18n) 字典支持。
  - `modules/ui.ts`: **交互**。实现 HITL 审批弹窗 (Shadow DOM)、自动发送、输入框回填 (保持原生 DOM 操作以减少体积)。
  - `modules/config.ts`: **配置**。集中管理各 AI 平台的 DOM 选择器。
- **持久化**: `background/index.ts` 负责在插件更新时保护用户配置不被覆盖，并处理安全熔断。

#### 2. 交互界面 (Popup & Options)
- **Popup**: 三态视图（Connected / Available / Disconnected），支持一键连接。
- **Options (设置页)**:
  - **Sticky Footer**: 底部悬浮保存栏，防止配置丢失。
  - **HITL 管理**: 勾选敏感工具进行拦截。
  - **Prompt 管理**: 自定义 System/Training/Error Prompt。

## 3. 关键特性机制

### 🛡️ Human-in-the-Loop (HITL) 人工审批 (v2.1 Secure & Localized)
- **机制**: 
  - **拦截**: `content.js` 在执行工具前检查 `protected_tools` 名单。
  - **自动防御 (Auto-Protect)**: 首次发现新工具时，默认自动加入保护名单。
- **安全与交互 (Security & UX)**:
  - **防注入 (Sanitization)**: 对工具参数进行 HTML 转义，防止 XSS 攻击和布局崩坏。
  - **视图切换 (Inline View)**: 永久授权采用卡片内平滑切换，替代原生 Alert，体验统一。
  - **国际化 (I18n)**: 弹窗 UI 完全支持中/英双语自适应。
  - **Shadow DOM**: 样式隔离，互不干扰。
- **流程**: 
  - **批准 (Approve)**: 单次放行。
  - **永久允许 (Always Allow)**: 二次确认后将工具从保护名单移除。
  - **拒绝 (Reject)**: 支持填写拒绝理由。

### ⚡ 并发批处理队列 (Concurrency Queue)
- **痛点**: 解决 AI 一次性调用多个工具导致的回填乱序或漏发问题。
- **逻辑**: `resultBuffer` 暂存所有结果 -> 等待当前轮次 **所有** 工具完成 -> 按 DOM 顺序拼接 -> 一次性回填。

### 🔄 健壮的自动发送 (Robust Auto-Send)
- **焦点抢占**: 强制 `inputEl.focus()`，绕过浏览器后台限制。
- **事件模拟**: 触发 `input/change` 事件适配 React/Vue 框架。

### 📉 工具分层发现 (Tiered Tool Discovery)
- **机制**: 网关将工具分为 **Hot (基础)** 和 **Cold (扩展)** 两类。
- **流程**: 
  1. `list_tools`: Hot 工具返回全量 Schema；Cold 工具返回 `[Schema Hidden]` 描述。
  2. **Query**: AI 识别到隐藏标记，调用 `get_tool_definitions(names)`。
  3. **Execute**: 获得完整定义后，AI 发起真正的工具调用。

## 4. 当前状态
- **版本**: v0.5.0 (Monorepo Engineering).
- **最近更新**:
  - ✨ **Core**: 新增内置工具 `run_in_terminal`。支持在 VS Code 前台终端执行长耗时任务（如 `npm start`），解决后台进程超时问题，并提供实时日志可见性。
  - 💤 **Gateway**: 新增 30 分钟自动休眠机制 (Auto-Shutdown)，释放系统资源。
  - 🔗 **Gateway**: 实现会话持久化，重启服务不再重置 Token，避免重复连接。
  - 💾 **Config**: 新增配置导入/导出功能，解决插件 ID 变更导致白名单丢失的问题。
  - 📝 **Protocol**: 工具调用新增 `purpose` 字段，强制 AI 解释操作意图（CoT）。
  - 📉 **Performance**: 实现 **工具懒加载 (Tool Lazy Loading)**。`list_tools` 默认隐藏低频工具（如 Git）的 Schema 以节省 Token，AI 需通过 `get_tool_definitions` 按需获取。
  - 🎨 **UX**: 工具调用状态可视化升级（🔵处理中 -> 🟢成功 / 🔴错误）。
  - 🎨 **UX**: HITL 审批弹窗新增 `Purpose` (意图) 展示字段，提升审批效率。
  - 🎨 **UX**: 设置页增加悬浮保存栏；移除原生 Alert 改为卡片内视图切换。
  - 🛡️ **Security**: 修复 HITL 弹窗 XSS 漏洞；限制超长参数显示。
  - 🌏 **I18n**: 授权弹窗全面支持中文。
  - ⚙️ **Configuration**: 将 `mcpGateway.servers` 配置中的 `enabled: boolean` 字段替换为 **`disabled: boolean`**。现在只有显式设置为 `true` 时才会禁用服务，默认为启用状态。
  - 🏗️ **Engineering**: 完成 Monorepo 改造，引入 `@webmcp/shared` 共享模块；浏览器插件升级为 Vite + TypeScript 架构。

- 🐛 **Bridge**: 修复自动发送竞态条件 Bug (Auto-Send Race Condition)。引入双重防护：新任务中断发送 (Cancel Auto-Send) 与 Stop 按钮状态检查（已适配 DeepSeek SVG/AI Studio Spin/Gemini Fuzzy），移除了解析层死锁风险。
- ☁️ **Host Sync**: 实现配置云同步 (Host Sync)。浏览器插件连接时自动从 VS Code 拉取配置，保存时自动推送备份。配置持久化于 VS Code 的 `globalState`，解决开发环境下插件重装导致配置丢失的痛点。
- 📝 **User Rules**: 新增用户自定义规则配置。用户可定义个性化偏好（如“总是先写方案”），该规则会自动追加到初始 System Prompt 和周期性 Training Hint 中，并支持云同步。
- ⚡ **Performance**: 性能优化。将浏览器插件的 DOM 检测机制从定时轮询 (`setInterval`) 重构为事件驱动 (`MutationObserver`)，显著降低静止状态下的 CPU 和电量消耗。
