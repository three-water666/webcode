# 🧠 Project Context & Memory (Final State)

## 1. Project Overview (WebMCP v2.0 Native)

**Goal**: Connect Web-based AI to Local VS Code via Puppeteer (CDP).

### 🏗 Structure
- **`/mcp-gateway-vscode`**: 
  - `extension.ts`: Entry point.
  - `browserManager.ts`: **Core Logic**. Controls Chrome, injects scripts, manages WebSocket bridge.
  - `gateway.ts`: Connects to local MCP servers.
- **Legacy**: `mcp-bridge-browser` folder is deprecated and can be deleted.

## 2. Key Features Implemented
- **Visual Persistence**: Green 'Done' borders persist across re-renders (Global Scan logic).
- **Batch Processing**: Parallel execution of multiple tool calls.
- **Smart Retry**: Auto-retry logic for 'Send' button.
- **Data Persistence**: Uses `~/.webmcp/browser_data` to store cookies and plugins.
- **Initial Prompt**: Auto-paste (but manual send) for new chats.

## 3. Development Norms
- **Browser Control**: Always use `puppeteer-core`. 
- **Injection**: Logic resides in `browserManager.ts` -> `page.evaluateOnNewDocument`.
- **Logs**: Browser logs are forwarded to VS Code Output channel.

## 4. Status
- **Release Ready**: v2.0 Native Mode is fully functional.
- **Action Item**: Delete legacy `mcp-bridge-browser` folder manually.
